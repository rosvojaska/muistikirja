setup

git init

Alottaa gittailun siinä kansiossa. Eli tällä aina aloitetaan.

git config --global user.name 'Jaska Numminen' git config --global user.email 'numminen.jaska@gmail.com'

Tälläin säädetään nää asetukset.

git status

Kertoo että missä mennään.

git add index.html

Valmistelee (stage) valitut tiedostot ja laittaa ne sinne valmistelualueelle. Voi käyttää valitsijoita kuten *.html jne.

git rm --cached index.html

Poistaa valitut tiedostot sieltä valmistelualueelta (eli staging area, missä ne staged filessit on).

git commit

Tallentaa kaikki valmistellut (staged files) tiedostot paikallisesti, mutta ei vielä tallenna niitä nettiin. Commited on siis pysyvästi muutettu suomeksi. Gitille pitää kertoa, että mitä tiedostoja sen pitää valmistella sillä $git add-komennolla.

Muista pistää tälle komennoks että -m "tekstiä" niin se määrittää sen commit-viestin.

git branch branchin nimi

Lisää uuden haaran, tai mikä nyt branch onkaan suomeks, tällä nimellä.

git checkout branchin nimi

Vaihtaa johonkin toiseen branchiin.

git merge branchin nimi

Yhdistää tän branchin siihen ylempään.

git remote add origin https://github.com/rosvojaska/nimi.git

Tää synkkaa sen netin kanssa sen paikallisen gitin.

git push

Pistää kaikki fileet mitkä on ensin valmisteltu (add) ja sitten pysyvästi muutettu (commit) sinne nettiin talteen.

touch .gitignore

Tää tekee sellasen .gitignore-tiedoston, mihin voi lisätä ne tiedostot jota ei halua lisättävän siihen gittiin. Kansiot tulee tyyliin /dir
